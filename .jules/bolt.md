## 2024-03-31 - BioNetGen AST Evaluation Bottleneck & Security Flaw
**Learning:** The `Expression::evaluate` module in `bng2/Perl2/Expression.pm` previously relied on dynamically building a Perl string representation of the Abstract Syntax Tree (AST) and executing it using `eval "..."`. This was executed in a tight recursive loop during network generation and simulation. String `eval` in Perl is extremely slow due to the need to invoke the compiler at runtime, causing a massive performance bottleneck. Furthermore, evaluating un-sanitized model expressions dynamically presents a significant Arbitrary Code Execution (ACE) vulnerability if malicious models are parsed.
**Action:** Replaced dynamic string `eval` with direct AST interpretation and native Perl operators. For unary math functions, preserved compatibility by evaluating the child node *outside* the `eval` string, then falling back to `eval "$operator(\$v)"` safely. Ensured short-circuit evaluation for `&&`/`||` and right-associativity for `**` were correctly replicated manually.
